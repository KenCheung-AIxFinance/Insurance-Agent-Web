# 🚶‍♂️《保險經紀人用戶旅程圖：As-is → To-be》


---

# 💥 **As-is（現況流程）– 無系統輔助的痛點旅程**

## 🧩 Persona

* **AIA/Prudential 經紀人 / Team Manager**
* 工作模式：依靠個人經驗 + 公司後台系統
* 主要痛點：保單難懂、需手動整理建議書、資訊分散


---

## 🌪 **As-is User Journey Map**

| 階段 | 行為 | 心情 | 痛點 |
|----|----|----|----|
| 1. 準備會面 | 手動從公司後台拉 proposal / 找舊檔 | 😐 | 系統分散、不同公司格式不同 |
| 2. 瞭解客戶需求 | 使用 Excel / 手寫做簡單 KYC、缺口分析 | 😕 | 無統一流程、資料散亂 |
| 3. 客戶詢問條款 | 打開 50+ 頁產品冊 PDF，Ctrl+F 慢慢找 | 😣 | PDF 冗長、用語艱澀、難快速回答 |
| 4. 條款解釋 | 將 PDF 內容自行濃縮為客戶版本 | 😵 | 易出錯、耗時、無統一話術 |
| 5. 生成建議書簡化版 | 手動 copy 多個 proposal、剪貼編輯 | 😢 | 格式不一致、反覆修改、工作量大 |
| 6. 投保流程講解 | 自行整理圖片、截圖、影片 | 😕 | 資訊零散、準備週期長 |
| 7. 會後跟進 | 手動傳文件給客戶 | 😐 | 無統一文檔庫、版本混亂 |

**總結痛點：**
✔ 多系統跳來跳去
✔ 條款太長、專業詞過深，客戶聽不懂
✔ 產出簡化建議書極度耗時
✔ 所有事都靠經驗、人力完成
✔ 團隊難建立統一標準流程


---

# 🌈 **To-be（導入本系統後的理想旅程）**

## 🎯 核心價值

* 將 **散裝工具 → 流程化專案式「客戶 Case」**
* 條款查詢一秒找到原文＋兩種解釋
* Proposal 一鍵生成客戶可理解的版本
* 投保流程統一標準化、圖文化


---

## 🌟 **To-be User Journey Map（導入後）**

| 階段 | 行為 | 心情 | 功能輔助 | 成果 |
|----|----|----|----|----|
| 1. 開 Case | 在系統中建立「客戶案」 | 🙂 | Case 建立頁 | 客戶資料集中管理 |
| 2. KYC/需求 | 開啟外部計算器 / 系統內 KYC 模組 | 🙂 | 第一步固定區（外部鏈接） | 資料流清晰 |
| 3. 生成 Proposal | 經紀人從公司後台下載原 Proposal → 上傳系統 | 🙂 | 建議書生成模組 | 格式統一準備好 |
| 4. 條款查詢 | 在聊天框輸入客戶問題 → 系統直接搜尋條款 PDF | 😄 | 條款檢索：原文＋專業解釋＋客戶語言版本 | 快速回答客戶、提升專業形象 |
| 5. 生成簡化版建議書 | 一鍵產生「客戶版 Proposal」 | 🤩 | 模板 + LLM 資料抽取 | 5 分鐘完成原本 2 小時工作 |
| 6. 投保流程講解 | 顯示固定模塊：資料、材料、銀行、步驟 | 🙂 | 預設教育頁 | 統一專業形象 |
| 7. 會後跟進 | 自動整理文件、保留在 Case 中 | 😌 | 文件庫、Case 歷史 | 紀錄完整，有據可查 |

# 🌟 To-be 旅程詳述（UX Designer 可直接使用）

以下以「系統導入後」的理想流程，用更詳細的旅程分解：


---

## **1. 開啟會面前 – Case Setup**

### 用戶行為

* 點擊「新增 Case」
* 輸入客戶姓名、需求類型（儲蓄/醫療/教育金）

### 系統輔助

* 自動生成 Case Folder
* 自動建立 3 個階段導覽：
  **① KYC → ② 建議書 → ③ 投保講解**

### 用戶感受

\-「終於有個像 CRM 的東西了，不用每次從零開始」


---

## **2. 第一階段 – KYC / 缺口分析**

### 用戶行為

* 點擊外部鏈接（教育金計算器/退休金計算）
* 填完後將數字記錄回 Case 的 “KYC Notes”

### 系統協助

* KYC Notes 記錄區
* 未來可擴展計算器


---

## **3. 第二階段 – 建議書生成**

### 用戶行為

* 將原 Proposal PDF/Excel 上傳系統
* 選擇模板（例如「教育金簡化建議」）
* 一鍵生成

### 系統執行

* 解析原 PDF → 抽取保費/收益/計劃名
* 套用模板 → 自動生成「客戶易懂建議書」

### 用戶感受

🤩「原本至少 30 分鐘剪貼 → 30 秒搞掂」


---

## **4. 條款查詢（會中）**

### 用戶行為

* 客戶詢問：「重大疾病有沒有豁免保費？」
* 經紀人輸入「豁免保費是什麼？什麼時候適用？」

### 系統執行

* 搜索條款 PDF → 找出對應句段
* 顯示：
  * **條款原文**
  * **專業解釋（經紀版）**
  * **白話文（客戶版）**

### 用戶感受

😄「我回應超快！客人覺得我好專業」


---

## **5. 第三階段 – 投保流程講解**

### 用戶行為

* 點擊固定模板：「投保流程」

### 系統呈現

* 四步視覺化卡片：
  **資料收集 → 材料 → 體檢/銀行 → 簽署流程**

### 用戶感受

😊「展示專業，不再手忙腳亂」


---

## **6. 會後跟進**

### 系統自動

* 所有文件、建議書、條款查詢紀錄都留在 Case 裡
* 經紀人可複製分享連結或匯出 PDF

### 用戶感受

😌「資料不會再散落 WhatsApp」


---

# 🎁 **總結圖：As-is → To-be**

      ↓
To-be:
