創建一個前端項目，後期將會與後端整合。使用react tsx vite tailwindcss 等作為基礎，設計一個簡單醫用，功能齊全的 RAG Agentic 助手，這是用於保險行業的 LLM Agent，主要功能是保險計劃書管理和查詢問答，和PPT 創作，這個 PPT 創作是允許上傳模板和數據源，然後 agent 負責聚合數據都這個模板裏面，形成一個簡化版的計劃書閱讀材料。

你的工作範圍是確保前端界面可以演示，具體功能並不需要實現，只需要確保美觀和實用的交互。

---
創建一個基於 React + TypeScript + Vite + TailwindCSS 的前端項目框架，專為保險行業的 RAG Agentic 助手設計。該項目需具備以下核心功能模塊的界面展示能力：

1. 項目基礎架構：

- 使用 Vite 構建工具配置 React + TypeScript 開發環境

- 整合 TailwindCSS 進行 UI 設計和樣式管理

- 預留後端 API 對接的接口配置

2. 主要功能界面設計：

2.1 保險計劃書管理模塊：

- 設計直觀的計劃書列表展示界面

- 創建計劃書表單的 UI 佈局

- 計劃書詳情查看頁面

2.2 智能問答模塊：

- 設計聊天式問答界面

- 包含消息歷史記錄展示區域

- 問題輸入框和發送按鈕

2.3 PPT 創作模塊：

- 模板上傳界面設計

- 數據源上傳區域

- PPT 生成預覽面板

- 下載按鈕和狀態顯示

3. 交互設計要求：

- 實現流暢的頁面切換動畫

- 設計統一的表單驗證提示

- 創建加載狀態指示器

- 錯誤提示和處理界面

4. 視覺設計規範：

- 採用醫療/保險行業風格的配色方案

- 確保界面元素間距和層次清晰

- 設計響應式佈局適應不同設備

- 創建統一的組件庫風格指南

5. 演示準備：

- 配置路由系統實現模塊間導航

- 創建模擬數據展示各功能界面

- 設計狀態管理結構（可預留 Redux 或 Context API 整合點）

- 確保所有交互元素有視覺反饋

注意：只需完成前端界面和基礎交互邏輯，無需實現實際業務功能，但要確保界面設計能清晰展示未來功能運作流程。
